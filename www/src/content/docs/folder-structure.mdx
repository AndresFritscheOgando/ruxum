### The `www/lib/api.ts` client

The full-stack scaffold includes a typed fetch utility pre-wired to your Axum API:

```ts
// www/lib/api.ts
const API_URL = process.env.NEXT_PUBLIC_API_URL ?? 'http://localhost:3000';

export async function apiFetch<T>(path: string, init?: RequestInit): Promise<T> {
  const res = await fetch(`${API_URL}${path}`, {
    headers: { 'Content-Type': 'application/json' },
    ...init,
  });
  if (!res.ok) throw new Error(`API error: ${res.status}`);
  return res.json();
}
```

Set `NEXT_PUBLIC_API_URL` in `www/.env.local` to point at your production API.

## What is not included

Ruxum deliberately omits things you should choose yourself:

- **Frontend state management** вЂ” use Zustand, Jotai, or React context as appropriate
- **CSS component libraries** вЂ” use shadcn/ui, Radix, or plain CSS
- **Testing setup** вЂ” use Vitest, Jest, or Playwright depending on your needs
- **Deployment configuration** вЂ” use the deployment guide for Vercel, Fly.io, or Docker
